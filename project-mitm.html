Project overview

This lab demonstrates how man‑in‑the‑middle attacks work in a controlled environment using ARP poisoning, Ettercap, and Wireshark. The goal is to understand both how attackers intercept traffic and how defenders can detect suspicious patterns and protect sensitive data with encryption.

Objectives

  • Build an isolated virtual network to safely perform ARP poisoning and traffic interception.
  • Use Ettercap to position an attacker between two hosts without their knowledge.
  • Capture and analyze intercepted traffic with Wireshark to identify sensitive data and attack signatures.
  • Discuss practical measures to prevent or detect MITM attacks in real networks.

Technical implementation

Virtual lab design. I created multiple virtual machines (attacker, victim, gateway) connected through an internal network to ensure that all experiments stayed inside the lab.

ARP spoofing. Using Ettercap, I performed ARP poisoning by sending forged ARP replies to convince the victim and the gateway that the attacker’s MAC address belonged to the other host.

Traffic interception. Once positioned as the man‑in‑the‑middle, I used Wireshark on the attacker machine to capture packets, inspect protocols, and see how unencrypted traffic could expose credentials or other sensitive information.

Detection. I examined ARP tables, duplicate IP warnings, and unusual ARP traffic in Wireshark to understand how network defenders can spot this type of attack.

Challenges and difficulties

  • Network configuration. One of the main difficulties was configuring the virtual network so that traffic actually flowed through the attacker machine. Misconfigured IP settings or routing rules often caused the network to break instead of allowing transparent interception.
  • Filtering useful data. Wireshark captured a huge amount of traffic, and at first it was hard to focus on relevant packets. I had to learn efficient display filters and techniques to quickly find HTTP requests, ARP packets, and potential credentials.

How I organized the work

I split the lab into preparation (network and VM setup), attack execution (ARP poisoning and interception), and analysis (Wireshark review and documentation). For each phase, I noted exact commands, screenshots of ARP tables, and Wireshark filters used, so the experiment is easy to repeat.

I also kept a small troubleshooting section in my notes where I documented common errors (no traffic passing, ARP tables not updating) and how I fixed them.

Key learnings

  • ARP is a simple, unsecured protocol, which makes it an easy target for spoofing and MITM attacks.
  • Unencrypted protocols expose sensitive information directly to any attacker able to intercept traffic.
  • Good monitoring and logging, combined with network segmentation and encryption, are essential to reduce the impact of MITM attacks.

Core activities

  • Building an isolated virtual network with attacker, victim, and gateway machines.
  • Launching ARP poisoning using Ettercap and observing changes in ARP tables.
  • Capturing and analyzing intercepted traffic with Wireshark.
  • Identifying patterns that could be used to detect a MITM attack.

Tools and technologies

  • Virtualization platform for lab isolation.
  • Ettercap for ARP spoofing and traffic redirection.
  • Wireshark for deep packet inspection and analysis.

Defensive takeaway

Performing these attacks in a lab environment made it easier to understand why enforcing encryption everywhere and monitoring ARP behavior are practical and necessary defenses in modern networks.
