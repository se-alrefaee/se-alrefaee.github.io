Project Overview

This file hashing project demonstrates expertise in data integrity verification and digital forensics. Hash functions are fundamental cryptographic tools used to detect unauthorized modifications, verify file authenticity, and maintain chain-of-custody in security investigations.

Through hands-on labs, I developed skills in implementing, analyzing, and comparing various hashing algorithms to ensure data integrity across systems and forensic investigations.

Hashing Algorithms Implemented

Cryptographic Hash Functions

  • SHA-256 (SHA-2 Family) - 256-bit hash, industry standard
  • SHA-512 (SHA-2 Family) - 512-bit hash, maximum security
  • SHA-3 - Latest NIST standard, Keccak algorithm
  • MD5 - Legacy algorithm (cryptographically broken)
  • BLAKE2 - Fast cryptographic hash function

Specialized Hash Functions

  • ssdeep - Fuzzy hashing for similarity detection
  • TLSH - Trend Micro Locality Sensitive Hash
  • CRC32 - Error detection (not cryptographic)
  • xxHash - Ultra-fast non-cryptographic hash

Password Hashing Functions

  • Bcrypt - Adaptive hashing with salt
  • Argon2 - Memory-hard password hashing
  • PBKDF2 - Key derivation with iterations
  • scrypt - GPU-resistant password hashing

Implementation & Techniques

File Hash Computation

Calculating cryptographic hashes for integrity verification:

import hashlib def calculate_file_hash(filepath, algorithm='sha256'): hash_obj = hashlib.new(algorithm) with open(filepath, 'rb') as f: for chunk in iter(lambda: f.read(4096), b''): hash_obj.update(chunk) return hash_obj.hexdigest() file_hash = calculate_file_hash('document.pdf') print(f"SHA-256: {file_hash}")

Hash Comparison & Verification

Verifying file integrity by comparing computed vs expected hashes:

def verify_file_integrity(filepath, expected_hash): computed_hash = calculate_file_hash(filepath) return computed_hash.lower() == expected_hash.lower() is_valid = verify_file_integrity('data.zip', 'expected_sha256_hash') print(f"File integrity verified: {is_valid}")

Bulk Hash Generation for Forensics

Processing multiple files for digital forensic investigations:

import os def generate_hash_manifest(directory): hashes = {} for root, dirs, files in os.walk(directory): for file in files: filepath = os.path.join(root, file) hashes[filepath] = calculate_file_hash(filepath) return hashes manifest = generate_hash_manifest('/evidence/')

Security Considerations

  • Avoid MD5 and SHA-1 - Both have practical collision attacks documented
  • Use SHA-256 minimum - Industry standard for new implementations
  • Include salt for passwords - Prevent rainbow table attacks
  • Use slow functions - Bcrypt/Argon2 add computational delay to brute-force
  • Implement rate limiting - Prevent hash-based attacks
  • Secure hash storage - Never transmit hashes over unencrypted channels
  • Document hash methods - Maintain forensic chain-of-custody
  • Regular updates - Stay informed of new hash vulnerabilities

Forensic Investigation Process

Step 1: Evidence Collection

Generate cryptographic hashes immediately upon evidence acquisition to establish baseline integrity.

Step 2: Hash Verification

Periodically recompute hashes to detect any unauthorized modifications during investigation.

Step 3: Similarity Analysis

Use fuzzy hashing (ssdeep, TLSH) to identify related files despite minor modifications.

Step 4: Database Comparison

Cross-reference hashes against known databases (NSRL, VirusTotal) to classify evidence.

Step 5: Documentation

Maintain detailed records of all hash operations for legal admissibility and chain-of-custody.
