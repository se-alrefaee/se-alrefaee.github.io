Project overview

This lab explores the fundamentals of cryptography through practical exercises with hashing, symmetric encryption, and asymmetric encryption. The focus is on understanding how these building blocks protect confidentiality and integrity in real systems, not only on running commands.

Objectives

  • Understand the difference between hashing, symmetric encryption, and asymmetric encryption.
  • Use tools to encrypt and decrypt files with shared keys and public/private key pairs.
  • Exchange encrypted messages with peers to simulate secure communication over an insecure channel.
  • Relate cryptographic concepts to real‑world use cases such as HTTPS, secure email, and file protection.

Technical implementation

Hashing. I generated hashes using MD5 and SHA‑256 to show how the same input always produces the same fingerprint and how tiny changes in a file produce a completely different hash.

Symmetric encryption. Using AES with a shared secret key, I encrypted local files and then decrypted them, verifying that only someone who knows the key can recover the original content.

Asymmetric encryption. I created RSA key pairs, shared public keys with classmates, and encrypted messages for each other that could only be decrypted with the corresponding private key.

Use cases. We discussed how these techniques are combined in protocols like TLS, where asymmetric encryption is used to exchange symmetric keys for performance reasons.

  • MD5 / SHA‑256
  • AES
  • RSA keys
  • OpenSSL / CLI tools

Challenges and difficulties

  • Key management. One difficulty was keeping track of which key belonged to which person and which key was public or private. I solved this by using a clear folder structure and naming convention for key files.
  • Understanding the workflow. At first, it was confusing to remember when to use a public key and when to use a private key, especially for signing versus encryption. Drawing small diagrams of the message flow helped make the process much clearer.

How I organized the work

I divided the lab into three blocks: hashing, symmetric encryption, and asymmetric encryption. For each block, I prepared a short checklist of commands to run, screenshots to capture, and questions to answer in the report.

I stored all commands in a script file and kept example outputs in a separate notes document, which made it easier to repeat the lab or adapt it to other tools later.

Key learnings

  • Hashing is about integrity and verification, while encryption is about confidentiality.
  • Symmetric encryption is fast and efficient but requires a secure way to share the key.
  • Asymmetric encryption simplifies key exchange and enables digital signatures, but is slower than symmetric algorithms.

Core activities

  • Generating MD5 and SHA‑256 hashes for different files.
  • Encrypting and decrypting example files using AES with a shared key.
  • Creating RSA key pairs and exchanging public keys with classmates.
  • Encrypting and decrypting messages using public/private key cryptography.

Tools and technologies

  • OpenSSL and command‑line encryption tools.
  • Linux shell for scripting repetitive tasks.
  • Text editors for managing keys and configuration files.

Practical impact

Completing this lab gave me a clearer mental model of how everyday technologies like HTTPS, VPNs, and secure storage rely on a combination of hashing, symmetric, and asymmetric cryptography.
