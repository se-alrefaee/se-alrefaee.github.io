Project overview

This project implements a realistic spear phishing simulation platform that combines LinkedIn data collection with AI‑generated, highly personalized emails for security awareness testing. It focuses on demonstrating the offensive capabilities of targeted phishing while highlighting the importance of detection, user education, and clear ethical boundaries.

Objectives

  • Automate the collection and structuring of publicly available LinkedIn profile information for testing purposes only.
  • Use AI models to generate personalized phishing emails that are significantly more convincing than generic templates.
  • Simulate end‑to‑end campaigns (collection, generation, sending, tracking) to evaluate an organization’s security awareness.
  • Document ethical considerations, legal limits, and technical safeguards for responsible use of such tools.

Technical implementation

Data collection. I designed a module that receives LinkedIn profile URLs, extracts public information such as job titles, current company, skills, and recent activity, and stores it as structured plain text for later processing.

AI email generation. The collected context is passed to an AI model which produces tailored spear phishing messages, such as fake job offers or meeting invitations, aligned with the target’s profile.

Campaign engine. Using the Gmail API, the system can send email batches, tag each campaign, and record basic engagement statistics like opens and apparent replies for later analysis.

Metrics and analysis. For each campaign, I track open rates, click indicators, and response behavior to compare personalized spear phishing results with generic phishing templates.

Security awareness use case. The platform is designed to be used in a controlled environment to help organizations identify at‑risk users and adjust training materials accordingly.

  • Python automation
  • HTTP requests
  • AI text generation
  • Gmail API
  • JSON data storage

Challenges and difficulties

  • LinkedIn anti‑bot protections. One major challenge was the anti‑automation and bot detection mechanisms on LinkedIn, which limited how aggressively I could collect profile data. I had to significantly slow down requests, randomize timings, and finally restrict myself to a small, controlled test set.
  • Project scope management. The original idea included a fully automated pipeline from large‑scale scraping to reporting dashboards, which turned out to be too ambitious for the time available. I decided to focus on a smaller but fully working prototype that covers a limited number of targets and core features only.

How I organized the work

I structured the project using Microsoft Planner, where I created separate buckets for data collection, AI prompt design, email sending, and reporting. Each task had a clear description, estimation, and deadlines so I could track progress and avoid getting stuck on one technical issue for too long.

I also maintained a simple changelog in the repository to document each important change in the scraping logic, prompt templates, and API integration, which made debugging and refactoring much easier.

Key learnings

  • Personalization drastically increases the realism and potential success of phishing emails, which makes user awareness and training critical.
  • Ethical and legal constraints must guide every design decision when working with real platforms and personal data.
  • Restricting the scope to a well‑defined prototype produces better, more stable results than trying to implement every idea at once.

Core features

  • Structured extraction of LinkedIn profile information for test targets.
  • Context‑aware AI prompt templates to generate realistic spear phishing messages.
  • Integration with an email API to automate campaign sending and tagging.
  • Basic metrics collection to compare campaigns and refine future simulations.

Tools and technologies

  • Python for automation and data processing.
  • REST APIs for LinkedIn (where allowed) and Gmail integration.
  • AI text generation models for email content.
  • GitHub for version control and project documentation.
  • Microsoft Planner for task management and planning.

Ethical considerations

The platform is strictly designed for controlled security awareness campaigns with prior authorization. All tests are limited in scope, conducted on consented accounts, and respect platform terms of service as much as possible.
